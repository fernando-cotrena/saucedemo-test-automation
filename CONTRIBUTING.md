# Contributing Guidelines

## Branching

- Always create a new branch before making any changes.
- Use descriptive names:
  - `feature/<feature-name>` for new functionality
  - `bugfix/<description>` for fixes
  - `chore/<description>` for configuration or docs

## Pull Requests

- Open a PR for review before merging into main
- Ensure all tests pass
- Add screenshots or reports if relevant
- Include a clear description of changes
